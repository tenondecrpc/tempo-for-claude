# Code of Conduct

## Purpose

This repository is maintained as a practical, respectful collaboration space for building Tempo. Contributors are expected to keep discussion technical, constructive, and focused on improving the project.

## Expected Behavior

Examples of behavior that help this project:

- Be respectful in issues, pull requests, reviews, and discussions.
- Assume good intent, but be precise when pointing out problems.
- Keep feedback actionable and tied to code, product behavior, or documentation.
- Welcome contributors with different experience levels and backgrounds.
- Accept maintainers' decisions about scope, quality, and release readiness.

## Unacceptable Behavior

The following behavior is not acceptable:

- Harassment, abuse, threats, or personal attacks.
- Discriminatory language or conduct.
- Trolling, baiting, or repeatedly derailing technical discussion.
- Publishing private, sensitive, or security-relevant information without permission.
- Spam, self-promotion, or low-effort noise that disrupts collaboration.

## Scope

This code of conduct applies to all repository spaces, including issues, pull requests, reviews, commit messages, and linked project discussions.

## Reporting

If you experience or witness behavior that violates this code of conduct, contact the repository maintainer through the contact options listed on the maintainer's GitHub profile:

- <https://github.com/tenondecrpc>

When reporting, include links, screenshots, and any context needed to understand what happened.

## Enforcement

The maintainer may remove comments, close discussions, reject contributions, or block participants whose behavior harms the project or its contributors.

Repeated or severe violations may result in a permanent ban from participation.
