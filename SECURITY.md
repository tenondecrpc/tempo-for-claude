# Security Policy

## Supported Versions

Security fixes are applied to the current development line only.

| Version | Supported |
| --- | --- |
| `main` | Yes |
| Older branches, forks, or snapshots | No |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Instead, report them privately to the repository maintainer through GitHub:

- <https://github.com/tenondecrpc>

If GitHub private vulnerability reporting is enabled for this repository, use that channel first.

## What To Include

Please include as much of the following as possible:

- A short description of the issue
- Affected component or target
- Reproduction steps or proof of concept
- Security impact
- Any suggested mitigation or fix

## Response Expectations

The maintainer will aim to:

- Acknowledge receipt within a reasonable time
- Reproduce and assess the report
- Fix the issue privately when possible
- Publish the fix after a patch is ready

Please avoid public disclosure until the maintainer has had a reasonable opportunity to investigate and respond.
