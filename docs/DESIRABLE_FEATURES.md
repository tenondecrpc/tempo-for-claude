# Desirable Features

Features observed in competing apps (primarily CUStats) that are intentionally **out of scope** for the `tempo-rebrand` change. These are candidates for future work.

## Out-of-Scope Feature List

1. **Multi-account support** — Ability to add and switch between multiple Claude accounts within a single app instance.

2. **Light mode** — Full light-mode theme support. Tempo currently enforces dark mode only.

3. **Pace prediction** — Predictive modeling of projected session/weekly usage based on current burn rate and historical patterns.

4. **Live session chart** — Real-time (sub-30s) animated chart updates during an active session.

5. **Day/week usage breakdowns** — Separate day-level and week-level breakdown views showing usage by weekday, hour-of-day heatmaps, or similar granular decompositions.

6. **Bar charts** — Alternative bar-chart visualization of usage alongside or replacing the line/area chart.

7. **Consumption rate histogram** — Distribution histogram showing how often usage reaches different utilization bands (0–25%, 25–50%, etc.) over time.

8. **Scheduled triggers / automations** — User-configurable automations (e.g., "send notification when session hits 80%") or integration with macOS Shortcuts.

9. **Codex / Claude API key support** — Support for tracking usage via a raw API key (as opposed to OAuth), for users accessing Claude through the API directly.

10. **All Accounts dashboard** — A consolidated view aggregating usage across multiple Claude accounts or workspaces.
